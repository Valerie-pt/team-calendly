# Запись на встречи — инструкция для команды

## Как это работает

Простой сервис для записи респондентов / кандидатов на встречи:

- В админке мы создаём **типы встреч** (например, «Customer Interview», «Кейс-интервью Boost»). У каждого типа — своя публичная ссылка и свой Zoom (при желании).
- Внутри каждого типа добавляем **слоты** — конкретные дата/время/длительность с указанием организатора.
- Респондент открывает ссылку конкретного типа, листает недели, выбирает свободный слот, оставляет имя/email/телеграм.
- После бронирования: респонденту приходит письмо с подтверждением, организатору и всем настроенным «notification emails» — уведомление о новой записи. В Google Calendar организатора (если он расшарил календарь на сервис-аккаунт) автоматически создаётся событие — и оба получают invite.
- Все данные хранятся в Google Таблице — листы `Events`, `Slots`, `Blocks`.

## Для команды: настройка нового типа встречи

В админке секция «Типы встреч»:

1. **Название** — что увидит респондент в шапке страницы (например, «Кейс-интервью Boost»)
2. **Slug** — URL-часть, генерируется из названия автоматически, можно поправить вручную
3. **Zoom-ссылка** *(необязательно)* — если оставить пустой, используется дефолтная из `NEXT_PUBLIC_ZOOM_LINK`
4. **Уведомления на email** — список адресов через запятую. Каждому из них (плюс организатору слота) будет приходить письмо при новой записи. Дублей нет — если организатор слота уже в списке, ему придёт одно письмо.

Поле «Уведомления» можно отредактировать у уже созданного типа: иконка ✎ рядом с типом.

## Для команды: добавление слотов

В секции «Слоты» (доступна, когда есть хотя бы один тип):

1. Выбери тип встречи в селекторе
2. Заполни своё имя и email (это интервьюер)
3. Поставь дату, время по МСК и длительность
4. Нажми «Добавить слот»

Имя и email сохраняются между добавлениями — можно быстро накидать несколько слотов подряд.

### Правила слотов

- **Время уникально по всему календарю.** Нельзя создать два слота на одну и ту же дату/время в разных типах встреч (потому что Zoom один на всех). Если время занято — система выдаст ошибку с указанием, где конфликт.
- **Прошедшие слоты скрываются** в админке автоматически.
- **Свободные слоты** видны в первой части секции, **забронированные** — в отдельной секции ниже (с данными кандидата: имя, email, Telegram).

### Удаление

Свободные слоты удаляются кнопкой «Удалить» в карточке. Забронированные слоты удалять нельзя через UI — если нужно отменить, удали запись напрямую в Google Sheets.

## Заблокированное время

Когда Zoom-аккаунт занят другой встречей вне системы — заблокируй интервал, чтобы коллеги случайно не создали слот на пересекающееся время.

В секции «Заблокированное время»:

1. Дата, время начала, длительность (15 мин — 8 часов)
2. Описание *(необязательно)*
3. Чекбокс **«Повторять каждую неделю в этот же день»** — если отмечен, блокировка работает на каждую такую же неделю-день навсегда

Заблокированное время:
- Не показывается респондентам
- Не даёт создавать слоты, пересекающиеся с ним
- Снимается кнопкой «Снять» в карточке

## Для респондентов: что отправлять

Отправляй ссылку конкретного типа встречи:

```
https://team-calendly.up.railway.app/{slug}
```

Например, для Кейс-интервью Boost:
```
https://team-calendly.up.railway.app/case-boost
```

Что увидит респондент:
1. Шапка с названием типа встречи
2. Указание часового пояса (МСК) и правила бронирования (не позднее, чем за 3 часа)
3. Слоты сгруппированы по дням и **листаются по неделям** (стрелки «‹» / «›»). Авто-открывается ближайшая неделя со свободными слотами.
4. Клик на слот → модалка: имя, email, Telegram *(необязательно)*
5. После бронирования — Zoom-ссылка, кнопки «Добавить в Google Calendar» и «Скачать .ics»

Респондент получает письмо-подтверждение. Команда (организатор слота + все, кто в notification emails этого типа) получают уведомление с данными респондента.

---

## Правила бронирования

- **Не позднее, чем за 3 часа** до начала. Слоты, до которых меньше 3 часов, не показываются и не бронируются.
- **Двойное бронирование невозможно** — при попытке забронировать одновременно с другим респондентом второй увидит ошибку «Slot is no longer available».
- **Слот пропадает у респондентов** сразу после бронирования (пока он не появится снова — а он не появится, если только админ не освободит его в таблице).

---

## Часовой пояс

Всё время на сайте — **по Москве (MSK, UTC+3)**. Это указано на каждой странице.

Когда респондент добавляет событие в свой Google Calendar / .ics — оно автоматически конвертируется в его локальный часовой пояс.

## Google Calendar для организаторов (опционально)

Если организатор расшарил свой календарь на сервис-аккаунт `team-calendly@team-calendly.iam.gserviceaccount.com` (с правами «Внесение изменений в мероприятия»), при бронировании в его календаре автоматически создаётся встреча с приглашением для респондента.

Если календарь не расшарен — приложение всё равно работает, просто событие в календаре приходится создавать вручную (или через ссылку «Добавить в Google Calendar» из письма).

## Bulk-загрузка слотов (для разработчика)

Когда нужно одной командой залить десятки слотов (например, после согласования общих свободных окон с несколькими организаторами):

1. Отредактируй `scripts/bulk-add-slots.ts` — укажи `EVENT_SLUG`, `INTERVIEWER_NAME`, `INTERVIEWER_EMAIL` и список дат/времён в МСК.
2. Запусти локально: `npx tsx scripts/bulk-add-slots.ts`

Скрипт находит событие по slug, читает существующие слоты (дубликаты по `date+time` пропускает) и аппендит новые в лист `Slots`.

## Вопросы и проблемы

- **Слот не появляется после добавления** — обновите страницу (F5).
- **Респондент не видит слоты** — проверь, что они в статусе «available», что дата ещё не прошла, и что до начала больше 3 часов.
- **Не приходят письма** — EmailJS на бесплатном тарифе ограничен 200 писем/месяц. Каждое бронирование = 1 письмо респонденту + по 1 на каждый адрес из «notification emails» + организатору. Считай.
- **Не создаётся событие в календаре** — у конкретного организатора, скорее всего, не расшарен календарь на сервис-аккаунт. Это не блокирует бронирование, только отключает авто-инвайт.
- **Не пускает в админку** — проверь `ADMIN_PASSWORD` в Railway → Variables и нажми «глазик» в форме логина, чтобы убедиться, что нет опечаток.

## Структура Google Sheets

| Лист | Колонки |
|------|---------|
| `Events` | id, name, slug, zoom_link, created_at, notification_emails (через запятую) |
| `Slots` | id, interviewer_name, interviewer_email, date, time, duration_minutes, status, candidate_name, candidate_email, candidate_telegram, event_id |
| `Blocks` | id, date, time, duration_minutes, recurring (TRUE/FALSE), label, created_at |

Сервис-аккаунт `team-calendly@team-calendly.iam.gserviceaccount.com` должен иметь Editor доступ к таблице.
